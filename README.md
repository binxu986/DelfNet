## ⚠️ DNN 现存问题

### 📉 训练问题

| 问题 | 描述 |
|------|------|
| 🔴 训练效率低下 | 神经网络层之间没有直接的联动 |
| 🔴 梯度传播局限 | 每一层的 dy 仅作用于当前层的权重 |

### 🏗️ 结构问题

| 问题 | 描述 |
|------|------|
| 🔴 网络过深 | 大脑并不一定需要这么多“独立的”参数层，现在 DNN 网络不得不深也侧面说明了效率不高 |
| 🔴 缺乏局部交互 | 神经元之间没有形成局部网络，同一层内的神经元没有联系，局部神经元之间没有进行充分的信息共享与竞争（竞争应导致激活与抑制等不同效果）。而例如 SOM(Self-organizing map)与 Hopfield 网络强调了一组神经元内部的信息交换 |
| 🔴 稀疏激活路径难发现 | 训练时前后层独立更新（本层 dy 仅作用于本层），难以找到有效的稀疏激活路径 |

### 🚀 推理问题

| 问题 | 描述 |
|------|------|
| 🔴 带宽瓶颈 | Decoding 需要每次都加载完整权重，效率太低；这个问题严重限制了当前 DNN 推理场景的应用效率 |

## 🔧 核心方案

### 📐 层级结构


```
Layer0 { Block0, Block1, …, Block4 }
Layer1 { Block5, Block6, …, Block9 }
…
```

### 🔑 关键技术：Delta Weight

用 **△ weight** 表达后续 Block 对前序 Block 权重的变化：

```
w1 = w0 + △1
w2 = w1 + △2 = w0 + △1 + △2
…
```

✨ **核心优势**：更新后续 Block 的变量时，dy 会直接作用于前序 Block 的变量，产生**联动效果**。

## 🧪 模型假设

| 假设 | 说明 |
|------|------|
| 🟢 △ 变化平滑 | △ 的变化不应该是随机、跳跃、完全自由的。Layer 内的 △ 序列变化应该是平滑的 |
| 🟢 △ 稀疏性 | △ 可能是稀疏的。Block 之间的联动更新，赋予了 △ 更容易被控制成稀疏的特性。△ 表示对输入信号的抑制或增强 |
| 🟢 △ 生成方式 | △ 可能受离线训练影响，也可能受前序 Block 在线激活情况影响（Data Dependent）。通过在线线性计算得到，既保留权重参数自由度，又不必离线保存，对 decoding 推理场景开辟新机会——**部分参数在线生成，缓解 bandwidth bound 问题** |

> 由近及远

| 阶段 | 目标 | 描述 |

| **Phase 1** | 🔨 基础改造 | 将 Transformer 改造成 DelfNet 形式，完成训练。解决过程中遇到的问题 |
| **Phase 2** | 📊 约束研究 | 研究 △ 之间的平滑约束、稀疏约束等假设，同等参数下做出 scaling law 更好的实验结果 |

| **Phase 4** | 🚀 算存融合 | 假设 △ 由 data 产生，以算代存，实现网络自适应数据的能力 |
